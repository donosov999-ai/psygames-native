## 1. Что это и где живёт

**PsyGames** — кроссплатформенное приложение когнитивных тренировок. 62 игры (память, внимание, логика, контроль, счёт, скорость) + уровни/боссы/геймификация.

| | |
|---|---|
| **Репо (единственный источник)** | `donosov999-ai/psygames-native` (private) |
| **Рабочая копия** | `~/dev/psygames` (реальная папка). `~/Downloads/Code claude/psygames` — **симлинк** на неё, не отдельный клон → рассинхрона быть не может |
| **Стек** | Expo Router + React Native 0.85 + react-native-web; Tauri 2 для десктопа |
| **Веб-деплой** | `psy-games.pro/play` (авто из CI, baseUrl `/psygames-web`) + `/downloads` |
| **Промо-сайт** | `psy-games.pro` — отдельный проект `~/dev/psygames-astro` (Astro, Contabo) |
| **Supabase** | `personal-nzt` (`iuvvheeocobhiothfgei`) — сессии, фидбек, словари |
| **Деплой** | push в `main` + тег `vX.Y.Z` → CI собирает Mac/Win/Android + деплой веба |

**Инфра фидбека тестировщиков (закрытый тест):**
- Кнопка в приложении → текст + скриншот → Supabase таблица `app_feedback` + бакет `feedback-shots` (приватный). В репорт пишутся: игра, уровень, профиль, ширина экрана, масштаб шрифта, реальная платформа (tauri-android/web).
- Пинг Денису: крон `*/2` на 89.116 (`/root/psygames_feedback_notify.py`) → RPC с хеш-гейтом → @Odv999_bot (текст: игра · уровень · профиль + скрин файлом).
- Скрины: Edge Function `feedback-shot` (service_role, гейт по sha256 секрета).
- **Просмотр Claude'ом со скринами:** `python3 ~/dev/psygames/_local/feedback_pull.py [N]` → тянет репорты + картинки локально → Read видит текст+скрин.
- Галочка «Чат с разработчиками» в настройках прячет кнопку у тестировщика (v1.125.0).
- ⚠️ getUpdates через токен @Odv999_bot НЕ работает (бот на webhook bots.asibots.pro) — читать чат бота напрямую нельзя.

## 4. Что висит — по приоритету

### 🔴 Блокеры (решение/действие за Денисом)
- [ ] **Apple Developer Program $99/год** — без него нет iOS-сборки. После оплаты → iOS CI job (сейчас в `build.yml` iOS нет вовсе).

### 🟠 Перед публичным релизом (обязательно)
- [ ] **`FEEDBACK_ENABLED = false`** в `appFeedback.ts` — иначе кнопка фидбека висит у всех, польётся мусор. Сейчас `true` для закрытого теста. (Галочка в настройках это НЕ заменяет — она по умолчанию ON.)
- [ ] **Приёмка волны фидбека на устройстве Rulon** — точечные правки игр (bart/hanoi/iowa раскладка, chess-blind фигуры, visual-search) и адаптивность крупного шрифта проверены статически + tsc + jest + живьём на главной, но каждую игру на его Android с 160%-шрифтом из сессии не прогнать.
- [ ] **Safe-area Android — добить если нужно.** `viewport-fit=cover` закрывает web/PWA/iOS. Если на Android без выреза лого всё ещё под статус-баром — патч в CI (решение за Денисом): **A** `windowOptOutEdgeToEdgeEnforcement` (~6 строк, умрёт на Android 16) · **B** инсет-листенер → CSS `--sat` (~25 строк Kotlin, переживёт Android 16) ← рекомендация · **C** костыль `Math.max(insets.top,24)`.

### 🟠 Каркас игр (GameShell) — ЗАФИКСИРОВАНО на будущее
Корень жалоб «кнопки то внизу, то в центре» — у 62 игр нет общего каркаса, каждая копирует layout (header×52, playArea×48). Владелец выбрал точечный путь (v1.124.0 — 5 игр), полный каркас отложен.
- [ ] **Полный `src/components/GameShell.tsx`** — обёртка playing-фазы: поле по центру + прибитый нижний тулбар действий (эталон math-sprint) + `scrollableField` для 10 скролл-игр. ~7ч постройка.
- [ ] **Миграция 62 игр когортами** со смоук-гейтом. ~30ч. Когорта 1 — ~40 кнопочных, когорта 2 — 10 скролл-полей.
- [ ] **Кнопка чата разработчиков → в тулбар** (перестаёт налезать на игру) + перетаскивание кнопки. Галочка отключения уже есть (v1.125.0).
- [ ] **Полезные кнопки в тулбар** (справка «?», пауза, рестарт) — сейчас «?»-FAB и чат-FAB висят глобально и пересекаются с полем.
- ⚠️ 2 решения по API до старта: тулбар прибитый футер vs кнопки-в-колонке; `scrollableField` проп vs отдельный компонент.
