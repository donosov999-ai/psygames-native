## ЛЕСЕНКИ (все 20 игр + 1 новая)

### Span-память (digit-span / spatial-span / corsi / ospan / reading-span)
Длина → потом скорость показа↑ (интервал↓) → обратный порядок (reverse) → помехи. Пример digit-span:
L1–6 длина 4→9 · L7–9 длина 7→9 быстрее показ · L10–12 reverse 5→7 · L13–15 reverse + быстрее. Аналогично spatial(span 2→7)/corsi(3→8)/ospan(setSize 3→7)/reading(3→7), дальше своя «выносливость» (скорость/reverse/сложнее операции-фразы).

### memory-matrix
L1–8 одна серия, сетка 3×3→6×6 + число вспышек↑ + скорость показа↑ · L9–12 **2 серии** (запомни A, потом B → воспроизведи обе — интерференция) · L13–15 2 серии **цветные** + порядок.
🔸 Решение: воспроизведение **раздельно по цвету** (серия A синяя / B жёлтая — видно, что удержал обе).

### n-back
L1–5 N=1→5 · L6–10 N=1→5 быстрее (ISI↓) · L11–15 **Dual n-back** (позиция + звук одновременно — классика Jaeggi).
🔸 Решение: после N=5 — **Dual n-back** (новая механика, честная нагрузка).

### hanoi
L1–4 3 стержня диски 3→6 · L5–9 **4 стержня** диски 5→9 · L10–13 **5 стержней** диски 9→12 · L14–15 +оптимум ходов (3⭐ за минимум).
🔸 Решение: максимум **5 стержней, если влезает по верстке; иначе откат на 4.** Проверить верстку на телефоне при кодинге.

### tower-london
⚠️ ИНСАЙТ (24.06, читал код): с 3 шарами (ёмкости [3,2,1]) пространство состояний мало → `minMoves` упирается в ~6–7, НЕ в 15. Значит просто «level→minMoves» не даёт 15 ступеней. Путь к 15: L1–6 minMoves 2→7 (3 шара) · L7–10 **4 шара** (ёмкости напр. [4,3,1]) · L11–15 **5 шаров** + скрыть цель через N сек (планирование по памяти) / лимит обдумывания. Требует обобщить `makePuzzle`/`bfsMin`/`PEG_CAPS` на N шаров (BFS depth поднять с 12). → ПЕРЕНОС в волну 2 (не чистая добивка).

### visual-search
Дистракторы↑ → число целей↑ → **конъюнктивный поиск** (цвет+форма) → скорость. 15 ступеней. (Первая волна.)

### math-sprint

### picture-pairs

### targets

### schulte (gated → персист-уровень)

### cpt (вигильность — БЕЗ времени как оси)

### pattern (числовые ряды — БЕЗ лимита времени)
Классы прогрессий по сложности: L1–2 арифметика +/− · L3–4 геометрия ×/÷ (и регрессии) · L5–6 степени/квадраты/кубы · L7–8 интегральные (накопит. суммы)/треугольные · L9–10 рекуррентные (Фибо/Люка/трибоначчи) · L11–12 переплетённые/знакочередующиеся (два правила) · L13 полиномиальные (разности 2–3 порядка) · L14 составные (aₙ=2aₙ₋₁+1) · L15+ спец «фрактальные» с ОДНОЗНАЧНЫМ next (простые, факториал, look-and-say, Морс-Туэ).
🔸 Расширить генератор `makeSequence()` в `pattern.tsx` (банк правил = функции «дай следующий»).
🔸 **Кнопка «💡 Подсказка»** (как «Подсказка 0/1» в судоку): 2 ступени — тап1 класс прогрессии, тап2 суть/формула. У каждого правила добавить `{label, hint}` RU/EN. Использовал подсказку → потолок **2⭐** за уровень. (дефолт, ждёт финального ок Дениса).
🔸 Фрактальные без детерминированного next — выкидывать.

## СИНЕРГИЯ ИГР (обсуждаем — идея Дениса 24.06, НЕ кодить пока)
Родственные игры «перетекают» друг в друга → task-switching (когнитивная гибкость, отдельный навык) + передышка через контраст модальности между уровнями. Слой ПОВЕРХ уровней (делать ПОСЛЕ волн — нужна глубина 15 ступеней в играх).

**Связки (родственные семейства):**
- Память вербальная↔пространственная: digit-span → corsi → spatial-span (переключение модальности).
- Сканирование→поиск: schulte → visual-search.
- Планирование: hanoi ↔ tower-london (почти одна игра, естественные ступени).
- Раб. память обновление→манипуляция: n-back → ospan → reading-span.
- Торможение/конфликт: go-no-go → stroop → cpt.

**Форматы перетекания (лёгкий→амбициозный):** (1) передышка-контраст — после тяжёлого уровня A короткий мини-раунд родственной B; (2) эстафета-трек — кампания семейства по нарастанию, общие очки; (3) комбо/босс — каждый 5-й уровень чередует A/B; (4) dual-task — две задачи параллельно (=dual n-back на максимуме).

ОТКРЫТО к обсуждению: какой формат основной + группировка по семействам. Вернуться после волн 1–4.
